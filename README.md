# Recipe Assistant

A conversational recipe chatbot powered by **Google Gemini** + **FAISS RAG** with a **Streamlit** UI.

---

## Features

- 🔍 **RAG pipeline** — Semantic search over 39,000+ recipes using local FAISS vector index
- 💬 **Conversational history** — Gemini remembers the last 10 turns of your conversation
- 🧠 **Smart filtering** — Automatically detects ingredient and step constraints (e.g. *"chicken tenders, fewer steps"*)
- 🎨 **Premium dark UI** — Glassmorphism design with streaming responses

---

## Quick Start

### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. Set your Google AI Studio API key
Create a `.env` file:
```env
GOOGLE_API_KEY=your_key_here
```
Or paste the key directly in the sidebar when the app starts.

### 3. Build the FAISS index (one-time, ~5–10 minutes)
```bash
python ingest.py
```
This reads `data/recipes_raw_nosource_ar.json`, embeds all recipes with `all-MiniLM-L6-v2`, and saves the index to `faiss_index/`.

### 4. Launch the chatbot
```bash
streamlit run app.py
```

## Example Queries

| Query | What happens |
|---|---|
| *"Suggest a recipe with chicken tenders that takes fewer steps"* | Smart filter: `ingredient_filter=["chicken tenders"]`, `max_steps=6` |
| *"What are some vegetarian pasta dishes?"* | Semantic search, no filters |
| *"What are the ingredients for that last recipe?"* | Conversation history used — Gemini recalls previous context |
| *"Can I substitute butter with olive oil in that?"* | Follow-up understood via history |

## Architecture

```
User Query
    │
    ▼
[Streamlit UI]  ←── conversation_history (session_state)
    │
    ▼
[Smart Filter Parser]  →  ingredient_filter, max_steps
    │
    ▼
[FAISS Retriever]  →  Top-k semantically similar recipes
    │
    ▼
[Gemini 1.5 Flash]  ←  system_prompt + history + RAG context + query
    │
    ▼
[Streamed Response]  →  Streamlit chat bubble
```
